# InboxOS — AI Email Command Center

InboxOS is an AI-powered email command center that connects Gmail, classifies messages with Groq/Gemini, and surfaces actionable insights across dashboard views (action items, bills, follow-ups, security, and more).

Original Figma design: https://www.figma.com/design/IvYUNDCSwTwJH1eqHBFwpU/Build-this-feature

## Prerequisites

- Node.js 20+
- pnpm
- A Supabase project with Google OAuth enabled
- Gmail API scopes configured on the Google OAuth client

## Quick start

```bash
pnpm install
pnpm dev
```

Open the URL printed by Vite (typically http://localhost:5173).

## Environment & secrets

### Supabase Edge Function secrets

Set these in **Supabase → Edge Functions → Secrets** (or Figma Make Settings → Secrets):

| Secret | Required | Description |
|--------|----------|-------------|
| `GROQ_API_KEY` | One of Groq/Gemini | Email classification |
| `GEMINI_API_KEY` | One of Groq/Gemini | Insights, chat, daily brief |
| `GOOGLE_CLIENT_ID` | For token refresh | Same client ID as Supabase Google OAuth |
| `GOOGLE_CLIENT_SECRET` | For token refresh | Google OAuth client secret |
| `ALLOWED_ORIGINS` | Production | Comma-separated CORS origins, e.g. `https://app.example.com,http://localhost:5173` |

`SUPABASE_URL`, `SUPABASE_ANON_KEY`, and `SUPABASE_SERVICE_ROLE_KEY` are provided automatically in the Edge Function runtime.

### Frontend config

Supabase project ID and anon key live in `utils/supabase/info.tsx` (autogenerated by Figma Make).

## Architecture

```
src/app/           React UI (lazy-loaded views)
src/lib/           Supabase client + normalizers
supabase/functions/server/   Hono edge API (Gmail + AI)
```

Google OAuth tokens are stored **server-side** in the KV store (`google_tokens:{userId}`), not in browser `localStorage`. The edge function refreshes expired tokens when `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET` are configured.

## Google OAuth setup (Supabase)

1. Enable Google provider in Supabase Auth.
2. Add redirect URL: `https://<your-app-origin>/`
3. Request offline access so Supabase returns a `provider_refresh_token`.
4. Add Gmail scopes in the Supabase Google provider settings or via the client `signInWithOAuth` scopes.

## Project structure

- `src/app/views/` — Feature screens (dashboard, inbox, AI assistant, etc.)
- `src/lib/normalize.ts` — Defensive parsers for AI/API responses
- `supabase/functions/server/google_tokens.tsx` — Server-side token storage & refresh
- `supabase/functions/server/index.tsx` — Gmail sync, classification, insights, chat
