# NudgeBox — Your inbox, gently nudged forward

NudgeBox is an AI productivity layer for Gmail. It finds replies, deadlines, bills, and follow-ups hiding in your inbox—then gives you a clear plan to handle them across dashboard views (action items, bills, follow-ups, security, and more).

Original Figma design: https://www.figma.com/design/IvYUNDCSwTwJH1eqHBFwpU/Build-this-feature

## Prerequisites

- Node.js 20+
- pnpm
- A Supabase project with Google OAuth enabled
- Gmail API scopes configured on the Google OAuth client

## Quick start (on your computer)

`localhost` in Chrome means **your Mac/PC**, not a cloud agent. You must start the dev server on the same machine as your browser.

```bash
# 1. Clone and enter the repo (if you haven't already)
git clone https://github.com/randyakuffo/BuildThisFeature.git
cd BuildThisFeature
git checkout cursor/address-review-callouts-9953

# 2. Install Node 20+ and pnpm, then:
pnpm install
pnpm dev
```

Vite will print a URL like `http://localhost:5173/` — open **that exact URL** in Chrome.

> **Using Cursor Cloud Agent?** The agent's server is remote. Either forward port **5173** in Cursor's **Ports** tab, or run `pnpm dev` locally in your own terminal (steps above).

### Chrome error → fix

| What you see in Chrome | Cause | Fix |
|------------------------|-------|-----|
| `ERR_CONNECTION_REFUSED` / "This site can't be reached" | Nothing is listening on port 5173 on your machine | Run `pnpm dev` in a terminal on your computer |
| `ERR_EMPTY_RESPONSE` | Dev server crashed or port forwarding dropped | Restart `pnpm dev`; in Cursor, re-forward port 5173 |
| Page loads but Google login fails | Supabase redirect URL missing | Add `http://localhost:5173/` in Supabase → Auth → Redirect URLs |
| Page loads but API calls fail (CORS) | `ALLOWED_ORIGINS` secret missing localhost | Set `ALLOWED_ORIGINS=http://localhost:5173` in Supabase secrets |
| Port already in use | Another app uses 5173 | Vite will auto-pick the next port — use the URL it prints |

### Don't have pnpm?

```bash
npm install -g pnpm
# or use npm directly:
npm install
npm run dev
```

### Supabase Auth redirect (required for Google login)

Add this URL in **Supabase → Authentication → URL Configuration → Redirect URLs**:

```
http://localhost:5173/
```

## Environment & secrets

### Supabase Edge Function secrets

Set these in **Supabase → Edge Functions → Secrets** (or Figma Make Settings → Secrets):

| Secret | Required | Description |
|--------|----------|-------------|
| `GROQ_API_KEY` | One of Groq/Gemini | Email classification |
| `GEMINI_API_KEY` | One of Groq/Gemini | Insights, chat, daily brief |
| `GOOGLE_CLIENT_ID` | For token refresh | Same client ID as Supabase Google OAuth |
| `GOOGLE_CLIENT_SECRET` | For token refresh | Google OAuth client secret |
| `ALLOWED_ORIGINS` | Production | Comma-separated CORS origins — **must include** `http://localhost:5173` for local dev |

`SUPABASE_URL`, `SUPABASE_ANON_KEY`, and `SUPABASE_SERVICE_ROLE_KEY` are provided automatically in the Edge Function runtime.

### Frontend config

Supabase project ID and anon key live in `utils/supabase/info.tsx` (autogenerated by Figma Make).

## Architecture

```
src/app/           React UI (lazy-loaded views)
src/lib/           Supabase client + normalizers
supabase/functions/server/   Hono edge API (Gmail + AI)
```

Google OAuth tokens are stored **server-side** in the KV store (`google_tokens:{userId}`), not in browser `localStorage`. The edge function refreshes expired tokens when `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET` are configured.

## Google OAuth setup (Supabase)

1. Enable Google provider in Supabase Auth.
2. Add redirect URL: `https://<your-app-origin>/`
3. Request offline access so Supabase returns a `provider_refresh_token`.
4. Add Gmail scopes in the Supabase Google provider settings or via the client `signInWithOAuth` scopes.

## Project structure

- `src/app/views/` — Feature screens (dashboard, inbox, AI assistant, etc.)
- `src/lib/normalize.ts` — Defensive parsers for AI/API responses
- `supabase/functions/server/google_tokens.tsx` — Server-side token storage & refresh
- `supabase/functions/server/index.tsx` — Gmail sync, classification, insights, chat
